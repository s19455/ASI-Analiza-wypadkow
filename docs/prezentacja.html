Analiza wypadków drogowych

Predykcja stopnia obrażeń

Artur Cichocki, Bartosz Pikutin, Wiktor Golba

Projekt zaliczeniowy ASI — PJATK

Dane: Montgomery County, Maryland (2015–2024)

Problem

Cel: na podstawie warunków wypadku (pogoda, prędkość, typ kolizji...) przewidzieć stopień obrażeń.

Dlaczego to ważne:

  • Planowanie bezpieczeństwa ruchu
  • Lepsze dysponowanie służb ratunkowych
  • Zrozumienie czynników ryzyka

3 klasy:

NO_INJURY — brak obrażeń (82%)

MINOR — drobne obrażenia (17%)

SERIOUS — poważne / zgon (1%)

Problem: silna nierównowaga klas — SERIOUS to tylko 1% danych.

Źródło: Montgomery County, MD — Crash Reporting (publiczny dataset)

Ważniejsze cechy:

  • Pogoda, oświetlenie, stan nawierzchni
  • Typ kolizji, ruch pojazdu
  • Limit prędkości
  • Typ pojazdu, zakres uszkodzeń
  • Data i godzina, lokalizacja GPS

Co zrobiliśmy z danymi:

  • Usunęliśmy kolumny z >90% braków
  • Imputacja medianą / modą
  • Nowe cechy: is_night, vehicle_age, crash_hour...
  • Label encoding (bo dużo kategorii)
  • Mapowanie 5 klas → 3 klasy

EDA — co zauważyliśmy

  • Wypadki SERIOUS częstsze w nocy i przy złej pogodzie
  • Wyższy limit prędkości → większe ryzyko poważnych obrażeń
  • Kolizje czołowe (HEAD ON) — najwyższy % SERIOUS
  • Szczyty wypadków: 7-9 i 16-18 (godziny szczytu)
  • Piątki — najwięcej wypadków

Najważniejsze cechy (wg Random Forest):

  1. Vehicle Damage Extent
  2. Collision Type
  3. Speed Limit
  4. Vehicle Movement
  5. Crash hour

Brakujące dane:

  • Related Non-Motorist — 97% (usunięte)
  • Off-Road Description — 91% (usunięte)
  • Reszta — poniżej 10%

Szczegóły w notebooks/01_baseline_model.ipynb

Pipeline ML (Kedro)

crash_data.csv | v [data_preparation] 5 węzłów usunięcie kolumn -> imputacja -> inżynieria cech -> mapowanie target -> enkodowanie | v [data_modeling] 3 węzły podział train/test -> trening modelu -> ewaluacja | v [tuning] porównanie 4 modeli + Optuna (50 prób)

Uruchomienie: kedro run albo kedro run --pipeline=tuning

Wizualizacja: kedro viz
